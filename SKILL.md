# Break Wit — 動画カット表自動生成スキル

## 概要

ユーザーが動画ファイルのパスを指定すると、シーン検出・スクリーンショット抽出・ナレーション文字起こし・OCRを自動実行し、カット表（PDF または PPTX）を生成する。

## トリガー条件

以下のいずれかに該当する場合、このスキルを起動する：

- ユーザーが動画ファイルのパスを指定した（拡張子: `.mp4`, `.mov`, `.avi`, `.mkv`, `.webm`, `.m4v`, `.mxf`, `.ts`）
- ユーザーが「カット表」「香盤表」「cut sheet」「break wit」などのキーワードを使った
- 動画ファイルに関連するカット表生成のリクエスト

## 環境変数

| 変数名 | 説明 | デフォルト |
|--------|------|-----------|
| `BREAKWIT_DIR` | Break Wit プロジェクトのルートディレクトリ | （必須） |
| `BREAKWIT_PORT` | サーバーポート | `3001` |

## 処理フロー

### 1. ユーザー意図の解析

メッセージから以下を判定する：

| 項目 | 判定方法 | デフォルト |
|------|----------|-----------|
| 出力フォーマット | 「PDF」「パワポ」「PPTX」「PowerPoint」等のキーワード | `pdf` |
| テンプレート | テンプレート名の直接指定、または「横置き」「3列」等の説明 | 自動選択（横動画→h-portrait-2col-12、縦動画→v-portrait-2col-12） |
| 台本テキスト | `.txt` ファイルパスが一緒に指定されているか | なし |
| しきい値 | 「細かく」「大まかに」等の表現 | `0.3` |
| スキップ | 「音声起こし不要」「OCRなし」等 | すべて有効 |

**フォーマット判定の例：**
- 「PDFで作って」「PDFにして」 → `pdf`
- 「パワポで」「PPTXで」「PowerPointで」 → `pptx`
- 指定なし → `pdf`

**テンプレート判定の例：**
- 「横置きで」 → `h-landscape-2col-12`
- 「3列で」 → `h-landscape-3col-21`
- 「縦動画」 → `v-portrait-2col-12`
- テンプレート名を直接指定（例: `h-landscape-3col-21`） → そのまま使用
- 指定なし → `h-portrait-2col-12`

### 2. セットアップ確認

以下を順に確認する：

1. `BREAKWIT_DIR` 環境変数が設定されていること
2. `$BREAKWIT_DIR/scripts/break-wit-cli.sh` が存在し実行可能であること
3. `$BREAKWIT_DIR/scripts/start-breakwit-server.sh` が存在し実行可能であること

```bash
# 検証コマンド
test -n "$BREAKWIT_DIR" && \
test -x "$BREAKWIT_DIR/scripts/break-wit-cli.sh" && \
test -x "$BREAKWIT_DIR/scripts/start-breakwit-server.sh" && \
echo "OK" || echo "NG"
```

検証に失敗した場合：

```
Break Wit の環境が正しく設定されていません。
1. export BREAKWIT_DIR=/path/to/cut-sheet-app を確認してください
2. $BREAKWIT_DIR/scripts/ にCLIスクリプトが存在するか確認してください
```

### 3. サーバー確認・起動

CLIを実行する前に、Break Witサーバーが起動しているか確認する：

```bash
bash "$BREAKWIT_DIR/scripts/start-breakwit-server.sh" --status
```

"stopped" と返ってきた場合は起動する：

```bash
bash "$BREAKWIT_DIR/scripts/start-breakwit-server.sh"
```

初回起動時はビルドが走るため1〜2分かかる場合がある。

### 4. CLI実行

シェルで以下のコマンドを実行する。タイムアウトは最低5分に設定すること。

**基本コマンド：**

```bash
bash "$BREAKWIT_DIR/scripts/break-wit-cli.sh" \
  --input "<動画ファイルのパス>" \
  --format <pdf|pptx> \
  --template <テンプレート名> \
  --output "<出力先パス>/<ファイル名>.<拡張子>"
```

**台本テキストがある場合：**

```bash
bash "$BREAKWIT_DIR/scripts/break-wit-cli.sh" \
  --input "<動画ファイルのパス>" \
  --format <pdf|pptx> \
  --template <テンプレート名> \
  --script "<台本ファイルのパス>" \
  --output "<出力先パス>/<ファイル名>.<拡張子>"
```

**出力ファイル名の命名規則：**
- 元の動画ファイル名をベースにする
- 例: `video.mp4` → `video_cutsheet.pdf`

### 5. 結果の処理

CLI は JSON を stdout に出力する。

**成功時の出力：**

```json
{
  "success": true,
  "outputPath": "/path/to/output.pdf",
  "cutCount": 15,
  "duration": 120.5,
  "resolution": "1920x1080",
  "format": "pdf"
}
```

**成功した場合、以下の情報をユーザーに報告する：**

```
カット表が完成しました！

カット数: 15
動画尺: 2分00秒
解像度: 1920x1080
フォーマット: PDF
出力先: /path/to/output.pdf
```

- `duration`（秒数）は「○分○秒」形式に変換して表示する

**エラー時の出力：**

```json
{
  "success": false,
  "error": "Error message"
}
```

**注意:** エラーJSONはstdout ではなく stderr に出力される場合がある。実行結果全体（stdout + stderr）を確認すること。

**エラーの場合：** ユーザーにわかりやすく伝える。

| エラー内容 | ユーザーへのメッセージ |
|-----------|---------------------|
| ファイルが見つからない | 「動画ファイルが見つかりませんでした。パスを確認してください。」 |
| FFmpeg関連 | 「動画の解析中にエラーが発生しました。別の形式で試していただけますか？」 |
| メモリ/ディスク不足 | 「サーバーのリソースが不足しています。しばらくしてから再度お試しください。」 |
| その他 | 「処理中にエラーが発生しました。もう一度お試しいただくか、別の動画でお試しください。」 |

## シーン検出の調整

| ユーザーの表現 | オプション |
|--------------|----------|
| 「もっと細かく」「カット多め」 | `--threshold 0.2` |
| 「大まかに」「少なめ」 | `--threshold 0.5` |
| 指定なし | デフォルト 0.3 |

## 処理スキップ

| ユーザーの表現 | オプション |
|--------------|----------|
| 「音声起こし不要」「ナレーションなし」「セリフ不要」 | `--no-narration` |
| 「テロップ検出不要」「OCRなし」 | `--no-ocr` |
| 「カットだけ」「映像のみ」「画だけ」 | `--no-narration --no-ocr` |

## 利用可能テンプレート一覧

### 横動画用
| テンプレート名 | 説明 |
|---------------|------|
| `h-portrait-1col-10` | 縦置き・1列・1ページ10カット |
| `h-portrait-2col-12` | 縦置き・2列・1ページ12カット（デフォルト） |
| `h-portrait-2col-14` | 縦置き・2列・1ページ14カット |
| `h-landscape-2col-12` | 横置き・2列・1ページ12カット |
| `h-landscape-3col-21` | 横置き・3列・1ページ21カット |

### 縦動画用
| テンプレート名 | 説明 |
|---------------|------|
| `v-portrait-2col-10` | 縦置き・2列・1ページ10カット |
| `v-portrait-2col-12` | 縦置き・2列・1ページ12カット |
| `v-landscape-1row-6` | 横置き・1行・1ページ6カット |
| `v-landscape-2row-6` | 横置き・2行・1ページ6カット |

## 会話ガイドライン

- **言語:** 日本語で応答する。ユーザーが英語で話しかけた場合は英語で応答する。
- **トーン:** カジュアルだが丁寧。映像制作の現場で使われるツールなので、プロフェッショナルな信頼感を保つ。
- **余計な説明は不要:** 処理状況と結果を簡潔に伝える。技術的な仕組みの説明は聞かれない限りしない。
- **リトライの提案:** エラー時は具体的な次のアクションを提示する。
