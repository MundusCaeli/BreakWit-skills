# Break Wit — 動画カット表自動生成スキル

## 概要

ユーザーが動画ファイルを送信すると、シーン検出・スクリーンショット抽出・ナレーション文字起こし・OCRを自動実行し、カット表（PDF または PPTX）を生成して返す。

## トリガー条件

以下のいずれかに該当する場合、このスキルを起動する：

- ユーザーが動画ファイルを送信した（拡張子: `.mp4`, `.mov`, `.avi`, `.mkv`, `.webm`, `.m4v`, `.mxf`, `.ts`）
- ユーザーが動画ファイルとともに「カット表」「cut sheet」などのキーワードを使った
- MIMEタイプが `video/*` のファイルが添付されている

## 処理フロー

### 1. ユーザー意図の解析

メッセージから以下を判定する：

| 項目 | 判定方法 | デフォルト |
|------|----------|-----------|
| 出力フォーマット | 「PDF」「パワポ」「PPTX」「PowerPoint」等のキーワード | `pdf` |
| テンプレート | テンプレート名の直接指定、または「横置き」「3列」等の説明 | 自動選択（横動画→h-portrait-2col-12、縦動画→v-portrait-2col-12） |
| 台本テキスト | 動画と一緒に `.txt` ファイルが送られているか | なし |

**フォーマット判定の例：**
- 「PDFで作って」「PDFにして」 → `pdf`
- 「パワポで」「PPTXで」「PowerPointで」 → `pptx`
- 指定なし → `pdf`

**テンプレート判定の例：**
- 「横置きで」 → `h-landscape-2col-12`
- 「3列で」 → `h-landscape-3col-21`
- 「縦動画」 → `v-portrait-2col-12`
- テンプレート名を直接指定（例: `h-landscape-3col-21`） → そのまま使用
- 指定なし → `h-portrait-2col-12`

### 2. 受信確認メッセージを送信

動画を受け取ったら、処理を開始する前に必ずメッセージを送る：

```
動画を受け取りました。カット表を生成します。
フォーマット: PDF / テンプレート: h-portrait-2col-12
処理中です...動画の長さによって数分かかる場合があります。
```

### 3. 環境検証

CLIを実行する前に、以下を確認する：

```bash
test -n "$BREAKWIT_DIR" && test -x "$BREAKWIT_DIR/scripts/break-wit-cli.sh" && echo "OK" || echo "NG"
```

検証に失敗した場合は「Break Wit の環境が正しく設定されていません。管理者に連絡してください。」と伝える。

### 4. サーバー確認・起動

Break Witサーバーが起動しているか確認する：

```bash
bash "$BREAKWIT_DIR/scripts/start-breakwit-server.sh" --status
```

"stopped" と返ってきた場合は起動する：

```bash
bash "$BREAKWIT_DIR/scripts/start-breakwit-server.sh"
```

初回起動時はビルドが走るため1〜2分かかる場合がある。

### 5. CLI実行

`exec` ツールで以下のコマンドを実行する。環境変数 `BREAKWIT_DIR` が Break Wit のプロジェクトディレクトリを指す。

**基本コマンド：**

```bash
bash "$BREAKWIT_DIR/scripts/break-wit-cli.sh" \
  --input "<動画ファイルのパス>" \
  --format <pdf|pptx> \
  --template <テンプレート名> \
  --output "<出力先パス>/<ファイル名>.<拡張子>"
```

**台本テキストがある場合：**

```bash
bash "$BREAKWIT_DIR/scripts/break-wit-cli.sh" \
  --input "<動画ファイルのパス>" \
  --format <pdf|pptx> \
  --template <テンプレート名> \
  --script "<台本ファイルのパス>" \
  --output "<出力先パス>/<ファイル名>.<拡張子>"
```

**出力ファイル名の命名規則：**
- 元の動画ファイル名をベースにする
- 例: `video.mp4` → `video_cutsheet.pdf`

**重要：** 動画の長さによっては処理に数分かかる。タイムアウトを十分に長く設定すること（最低5分）。

### 6. 結果の処理

CLI は JSON を stdout に出力する。

**成功時の出力：**

```json
{
  "success": true,
  "outputPath": "/path/to/output.pdf",
  "cutCount": 15,
  "duration": 120.5,
  "resolution": "1920x1080",
  "format": "pdf"
}
```

**成功した場合：**
1. `outputPath` のファイルをユーザーにファイルとして送信する
2. 以下の情報をメッセージに含める：

```
カット表が完成しました！

カット数: 15
動画尺: 2分00秒
解像度: 1920x1080
フォーマット: PDF
```

- `duration`（秒数）は「○分○秒」形式に変換して表示する
- トーンはカジュアルで簡潔に

**エラー時の出力：**

```json
{
  "success": false,
  "error": "Error message"
}
```

**注意:** エラーJSONはstdout ではなく stderr に出力される。`exec` の実行結果全体（stdout + stderr）を確認すること。

**エラーの場合：** ユーザーにわかりやすく伝える。技術的な詳細はそのまま出さない。

| エラー内容 | ユーザーへのメッセージ |
|-----------|---------------------|
| ファイルが見つからない | 「動画ファイルが見つかりませんでした。もう一度送っていただけますか？」 |
| FFmpeg関連 | 「動画の解析中にエラーが発生しました。別の形式で試していただけますか？」 |
| メモリ/ディスク不足 | 「サーバーのリソースが不足しています。しばらくしてから再度お試しください。」 |
| その他 | 「処理中にエラーが発生しました。もう一度お試しいただくか、別の動画でお試しください。」 |

### 7. コマンドが見つからない場合

`BREAKWIT_DIR` が未設定、またはCLIスクリプトが存在しない場合：

```
Break Wit の環境が設定されていません。管理者に連絡してください。
```

## シーン検出の調整

| ユーザーの表現 | オプション |
|--------------|----------|
| 「もっと細かく」「カット多め」 | `--threshold 0.2` |
| 「大まかに」「少なめ」 | `--threshold 0.5` |
| 指定なし | デフォルト 0.3 |

## 処理スキップ

| ユーザーの表現 | オプション |
|--------------|----------|
| 「音声起こし不要」「ナレーションなし」「セリフ不要」 | `--no-narration` |
| 「テロップ検出不要」「OCRなし」 | `--no-ocr` |
| 「カットだけ」「映像のみ」「画だけ」 | `--no-narration --no-ocr` |

## 利用可能テンプレート一覧

ユーザーがテンプレートについて質問した場合、以下を案内する：

### 横動画用
| テンプレート名 | 説明 |
|---------------|------|
| `h-portrait-1col-10` | 縦置き・1列・1ページ10カット |
| `h-portrait-2col-12` | 縦置き・2列・1ページ12カット（デフォルト） |
| `h-portrait-2col-14` | 縦置き・2列・1ページ14カット |
| `h-landscape-2col-12` | 横置き・2列・1ページ12カット |
| `h-landscape-3col-21` | 横置き・3列・1ページ21カット |

### 縦動画用
| テンプレート名 | 説明 |
|---------------|------|
| `v-portrait-2col-10` | 縦置き・2列・1ページ10カット |
| `v-portrait-2col-12` | 縦置き・2列・1ページ12カット |
| `v-landscape-1row-6` | 横置き・1行・1ページ6カット |
| `v-landscape-2row-6` | 横置き・2行・1ページ6カット |

## 会話ガイドライン

- **言語:** 日本語で応答する。ユーザーが英語で話しかけた場合は英語で応答する。
- **トーン:** カジュアルだが丁寧。映像制作の現場で使われるツールなので、プロフェッショナルな信頼感を保つ。
- **余計な説明は不要:** 処理状況と結果を簡潔に伝える。技術的な仕組みの説明は聞かれない限りしない。
- **リトライの提案:** エラー時は具体的な次のアクションを提示する。
